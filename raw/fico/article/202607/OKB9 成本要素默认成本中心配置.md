# OKB9 成本要素默认成本中心配置

## 核心用途

OKB9 用于 **为成本要素（会计科目）设置默认的成本中心**。

当 FI 凭证过账时，如果某个科目是成本要素但用户没有手动填写成本中心，系统会根据 OKB9 的配置自动带出默认成本中心，避免过账时报错"成本中心必填"。

## 典型业务场景

| 场景 | 说明 |
|------|------|
| **费用类科目过账** | 管理费用、销售费用等科目过账时自动带出默认成本中心 |
| **采购收货/发票校验** | 费用性采购订单过账时，自动分配成本中心 |
| **物料差异科目** | 不启用物料分类账时，通过 OKB9 将差异科目按公司代码分配到对应成本中心 |
| **GR/IR 科目** | 为应付暂估科目配置默认成本中心 |

## 操作步骤

**事务码：OKB9**

1. **进入 OKB9**，选择控制范围
2. **新建条目**，需要维护以下关键字段：

| 字段 | 含义 |
|------|------|
| **成本控制范围** | 如 3210 |
| **成本要素** | 需要配置默认值的会计科目（如 6602010100 职工薪酬） |
| **成本中心** | 默认分配的成本中心编码 |
| **有效起始日** | 配置生效日期 |
![[raw/assets/Pasted image 20260617103323.png]]
3. **保存**即可生效

## 优先级逻辑

OKB9 的默认成本中心优先级最低，以下方式会覆盖 OKB9 的设置：

1. **手工填写** → 用户在凭证中直接指定成本中心，优先级最高
2. **订单/WBS 结算规则** → 生产订单、内部订单、WBS 等成本对象优先
3. **替代规则（OBBH）** → 通过替代自动填写
4. **OKB9 默认值** → 以上都没有时，取 OKB9 的配置

## 实际应用场景

不启用物料分类账时，将差异科目设为成本要素，当差异发生时，通过 OKB9 将差异科目按照不同公司分配至相应的成本中心，月结时再将差异进行手工结转。

启用物料分类账后，差异科目一般改为资产负债表科目（表外科目），通过 CKMLCP 月末结算还原差异，就不再依赖 OKB9 了。

## 注意事项

- OKB9 配置是按 **控制范围 + 成本要素** 维度维护的
- 同一个成本要素可以按公司代码不同设置不同的默认成本中心
- 如果科目不使用默认成本中心（比如有多个承担部门），就不要在 OKB9 配置，强制要求用户手工填写更准确
