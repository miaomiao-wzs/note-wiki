## 导语

在 SAP S/4HANA 的销售流程中， **销售订单（Sales Order）** 是整个订单到现金（O2C）链路的 **中枢环节** 。它上承接售前活动（询价/报价），下启交货与开票，同时与物料管理（MM）、生产计划（PP）、财务会计（FI）深度集成。

但订单处理远不止"创建一张VA01"这么简单。在实际项目中，SD顾问需要处理以下场景：

•客户发来 **询价（Inquiry）** ，你给出 **报价（Quotation）** ，客户下 **标准订单（Standard Order）** ——这是最经典的售前→订单路径。

•客户签订了年度的 **数量合同或价值合同** ，后续通过 **释放订单（Release Order）** 分批下单。

•门店零售场景下，客户当场付款提货——使用 **现金销售（Cash Sales）** 。

•仓库发现破损需要补货或扣款——使用 **贷项凭证请求（Credit Memo Request）** 。

本文将系统梳理 **从询价到合同** 的完整订单处理体系，涵盖售前单据、标准销售订单、框架协议和特殊订单类型，并提供配置参考。

## 一、售前单据：询价与报价

### Presales Documents: Inquiry & Quotation

售前活动是O2C流程的起点。SAP S/4HANA提供两种标准的售前单据类型。

### 1.1 询价（Inquiry）— 凭证类型 IN

客户向你询问某个物料的价格或交货信息，你在系统中创建询价单记录。询价单可以 **不包含价格** （可关闭定价），主要用于记录客户意向。

| 特性 | 说明 |
| --- | --- |
| **凭证类别** | E（Inquiry） |
| **定价** | 按需 — 可配置为不执行定价 |
| **后续参考** | 可选 — 报价但可参照询价创建 |
| **完成规则** | **规则 A** — 一旦被参照即标记为已完成 |

### 1.2 报价（Quotation）— 凭证类型 QT

报价是向客户提供的 **具有法律约束力** 的价格和交货承诺。报价单包含完整的定价信息。

| 特性 | 说明 |
| --- | --- |
| **凭证类别** | J（Quotation） |
| **定价** | 完整执行定价过程 |
| **后续参考** | 可参照报价创建标准订单 |
| **完成规则** | **规则 B** — 仅当全部数量被参照时才标记完成，支持分批下单 |

> **最佳实践** ：询价不是强制步骤，可以从头创建报价。但使用询价→报价→订单的完整路径可提供更完整的 **销售漏斗分析** 数据。被拒绝的报价可以记录拒绝原因（如价格过高、交期太晚），为市场策略提供参考。

### 1.3 售前阶段的凭证流

```
询价(IN) ──→ 报价(QT) ──→ 标准订单(OR) ──→ 外向交货 ──→ 开票
   │             │
   └── 完成规则A ┘   └── 完成规则B ── 支持多次参考下单
```

> **关键配置** ：复制控制中，询价→报价使用 **完成规则A** （一次引用即完成），报价→订单使用 **完成规则B** （全部数量引用才完成）。此举确保报价可以被多次部分引用。

## 二、销售订单类型

### Sales Order Types

销售订单是SD中最核心的凭证类型。SAP标准系统预置了多种订单类型，覆盖不同的业务场景。

### 2.1 标准销售订单（OR）

**OR（Standard Order）** 是最通用的销售订单类型，适用于大部分常规销售场景。

核心控制参数回顾：

| 参数 | 配置值 | 业务影响 |
| --- | --- | --- |
| **凭证类别** | C（Order） | 订单类 |
| **后续交货类型** | LF | 外向交货 |
| **后续开票类型** | F2 | 标准发票 |
| **交货冻结** | 空 | 默认不冻结 |
| **开票冻结** | 空 | 默认不冻结 |
| **不完整性检查** | 激活 | 确保关键字段必填 |

### 2.2 退货订单（RE）

**RE（Returns）** 用于客户退货场景。退货订单触发退货交货（类型 RL），退货交货过账时使用移动类型 651（质检库存退货），不影响可用库存。

### 2.3 服务物料订单

对于服务类业务，SAP建议使用物料类型 **DIEN** ，其行项目类别组 **LEIS** 会自动确定不含物流行为的行项目类别。服务行项目可直接参照销售订单开票（不经过交货）。

## 三、框架协议：合同与计划协议

### Outline Agreements: Contracts & Scheduling Agreements

框架协议是长期合作客户的重要业务工具。SAP支持三种主流的框架协议类型。

### 3.1 计划协议（Scheduling Agreement）— DS

**计划协议（DS）** 是一种 **包含固定交货日期和数量** 的长期协议。与数量合同不同，计划协议直接在协议中输入计划行（交付排程）。

| 特征 | 说明 |
| --- | --- |
| **含计划行** | ✅ 是 — 直接在协议中输入交货计划 |
| **释放订单** | ❌ 不需要 |
| **典型场景** | JIT供货、长期框架下单 |

### 3.2 数量合同（Quantity Contract）— MK

**数量合同（MK）** 约定一个 **总数量** ，不包含计划行。客户通过 **释放订单（Release Order）** 参照合同创建标准订单，每次释放更新合同的已释放数量和剩余数量。

| 特征 | 说明 |
| --- | --- |
| **含计划行** | ❌ 不含 |
| **释放订单** | ✅ 需要 |
| **典型场景** | 年度框架协议，按需分批下单 |
| **剩余量跟踪** | 系统自动管理已释放/剩余数量 |

### 3.3 价值合同（Value Contract）— WK1 / WK2

**价值合同** 约定一个 **目标金额** ，客户在合同有效期内购买不超过该金额的货物或服务。

| 类型 | 凭证类型 | 说明 |
| --- | --- | --- |
| **通用价值合同** | WK1 | 支持多种物料和服务，项类别 WKN |
| **物料相关价值合同** | WK2 | 针对单一物料（如可配置物料），项类别 WKC |

价值合同的核心特性：  
\- 释放超过目标金额时，可配置系统响应（无反应/警告/错误）  
\- 支持 **开票计划** ，按多个日期和部分数量一次性开票  
\- 可设置 **授权释放合作伙伴** （规则A：客户列表；规则B：客户层级）  
\- 可通过 **品种模块（Assortment Model）** 或 **产品层次（Product Hierarchy）** 限制可购买的产品范围

### 3.4 释放订单的创建方式

释放订单是参照数量合同或价值合同创建的标准订单，支持四种创建方式：

1 **初始界面** — 选择"以参考创建"，选择合同

2 **凭证菜单** — 在凭证编辑中选择"创建参考→合同"

3 **事后分配** — 给现有订单分配合同编号

4 **自动搜索** — 系统自动查找开放的框架协议

## 四、特殊订单类型

### Special Order Types

在实际业务中，除了标准订单和框架协议，还有若干特殊场景需要专门的订单类型处理。

### 4.1 现金销售（Cash Sales）— CS

**现金销售（CS）** 用于客户当场付款、当场提货的场景（如零售门店）。

| 特性 | 现金销售（CS） | 标准订单（OR） |
| --- | --- | --- |
| 立即交货 | ✅ 是 | ❌ 否 |
| 即时开票 | ✅ 是 | ❌ 否 |
| 应收账款 | ❌ 无（记入现金账户） | ✅ 有 |
| 后续交货类型 | BV | LF |

保存现金销售时，系统自动创建交货类型 **BV** 并打印 **现金发票** 。发货过账后，金额直接记入现金科目，不产生客户应收款。

> SAP建议现金销售的发货过账通过后台程序自动执行，开票使用订单相关开票（Order-Related Billing），不开具纸质发票，以销售订单号作为财务凭证的参考。

### 4.2 急单（Rush Order）— SO

**急单（SO）** 适用于客户急需、需要立即从仓库提货的场景。与现金销售不同， **急单仍产生应收账款** ，不打印发票。

| 特性 | 急单（SO） |
| --- | --- |
| 立即交货开关 | ✅ 是 |
| 后续交货类型 | LF |
| 开具发票 | ❌ 不立即打印（统一开票） |
| 应收账款 | ✅ 有 |

保存急单时，系统自动创建标准外向交货（类型 LF），随后可进行拣配和发货过账。

### 4.3 贷项凭证请求（Credit Memo Request）— RE 相关

当客户投诉价格错误、货物破损或数量短缺时，使用 **贷项凭证请求** 进行处理。

处理流程：  
1\. 参照原开票凭证或销售订单创建贷项凭证请求  
2\. 系统自动设置 **开票冻结** ，等待审批  
3\. 负责员工审核后 **解除冻结**  
4\. 创建贷项凭证开票文档，自动生成会计分录（借：收入/贷：客户应收）

> 贷项凭证请求支持 **工作流审批** ：低于阈值自动释放，超过阈值推送至指定审批人。拒绝时记录拒绝原因供分析。

### 4.4 其他特殊订单

| 订单类型 | 凭证类型 | 说明 |
| --- | --- | --- |
| **免费交货** | FD | 寄送样品 |
| **后续免费交货** | SD | 因投诉补发，必须参照前序单据 |
| **寄售补货** | KB | 将货物发往客户寄售库存 |
| **寄售发货** | KE | 客户从寄售库存中提取，触发开票 |

## 五、订单处理最佳实践

### Sales Order Processing Best Practices

### 5.1 文档流监控

每个销售过程都会形成一条 **文档链** 。通过 **文档流（Document Flow）** 功能，可以：  
\- 一键查看完整处理历程（报价→订单→交货→开票→会计凭证）  
\- 查看每个节点的状态（交货状态、拣配状态、开票状态、过账状态）  
\- 直接跳转到任意前序或后续凭证

### 5.2 冻结控制策略

在销售订单中可以设置两种冻结：

| 冻结类型 | 作用层级 | 控制效果 |
| --- | --- | --- |
| **交货冻结** | 抬头/行项目 | 防止创建外向交货 |
| **开票冻结** | 抬头/行项目 | 防止创建开票凭证 |

> **最佳实践** ：贷项凭证请求默认设置开票冻结是保护性设计。同理，可在客户主数据中设置订单冻结（如"禁止免费交货"），在创建特定凭证类型时被系统拦截。

### 5.3 拒绝原因管理

为被拒绝的项目分配 **拒绝原因（Reason for Rejection）** ，系统自动标记为"已完成"状态。复制控制中的需求检查会阻止拒绝项目复制到后续凭证。拒绝原因数据可为销售策略分析提供有力支撑。

### 5.4 不完整性检查

在凭证类型配置中激活 **不完整性消息** 开关后，即使缺少部分非关键字段，订单仍可以保存。系统通过 **不完整性日志** 列出缺失字段，用户可直接跳转至对应字段补充。 **状态分组** 控制缺失字段对后续流程的影响（如某些字段缺失仅阻止开票、不阻止交货）。

### 5.5 销售订单履行监控（Fiori App）

SAP S/4HANA提供 **Sales Order Fulfillment** 应用程序，内部销售代表可以通过该应用：  
\- 查看所有 **无法履行的销售订单** 优先级列表  
\- 识别阻碍原因（信用冻结、ATP不足、价格缺失等）  
\- 一键采取补救措施（联系客户、调整订单、释放冻结）

## 结语

从 **售前单据** （询价IN、报价QT）到 **标准销售订单** （OR），再到 **框架协议** （计划协议DS、数量合同MK、价值合同WK1/WK2）和 **特殊订单类型** （现金销售CS、急单SO、贷项凭证请求），SAP S/4HANA为每一种销售场景都提供了"开箱即用"的凭证类型。

理解不同类型凭证的 **适用场景** 、 **行为差异** 和 **配置要点** ，是SD顾问从入门到进阶的必经之路。而 **文档流** 、 **冻结控制** 、 **不完整性检查** 和 **复制控制** 等机制，则为订单处理的健壮性和灵活性提供了制度保障。
