# SAP 如何批量扩充科目主数据到其他子公司

## 来源信息
- 来源类型：article
- 原始位置：`raw/fico/article/202607/SAP 如何批量扩充科目主数据到其他子公司.md`
- 原始来源：知乎文章 `SAP 如何批量扩充（复制）科目主数据到其他子公司|复制科目主数据详解 FS15&FS16`
- 语言：中文
- 适用模块：FICO

## 一句话结论
当新建子公司需要快速复制母公司或源公司代码的总账科目主数据时，可使用 `FS15` 直接复制或生成科目主数据文件，再结合 `CG3Y`、`CG3Z` 和 `FS16` 完成跨服务器或批量导入。

## 关键内容
- `FS15` 可将某公司代码的部分总账科目复制到其他公司代码。
- `FS16` 可通过上传文件的方式将科目主数据复制到目标公司代码。
- 批量扩充前，源公司代码和目标公司代码的科目组、容差组、字段状态组、留存收益科目等配置应保持一致。
- 如果使用文件方式，先用 `FS15` 生成源文件，再用 `CG3Y` 从应用服务器下载文件。
- 文件下载到本地后，可用 `CG3Z` 上传到目标应用服务器。
- 最后在目标系统或目标服务器使用 `FS16` 接收并导入科目主数据。

## 操作思路
1. 确认源公司代码和目标公司代码相关配置一致。
2. 使用 `FS15` 选择源公司代码、目标公司代码和科目范围。
3. 如为直接复制场景，执行后检查目标公司代码科目主数据。
4. 如为文件传输场景，使用 `FS15` 生成服务器源文件。
5. 使用 `CG3Y` 把服务器文件下载到本地。
6. 使用 `CG3Z` 把文件上传到目标服务器。
7. 使用 `FS16` 在目标端接收并导入科目主数据。
8. 导入后用 `FS00` 或科目清单检查目标公司代码科目是否完整。
