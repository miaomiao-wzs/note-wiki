# SAP S4HANA销售订单处理（从询价到合同）

## 一句话结论
销售订单是 SD 订单到现金流程的核心单据，上承询价、报价、合同等售前和框架协议，下接交货、开票和会计凭证；实际项目中要区分订单类型、复制控制、冻结控制、不完整性检查和文档流。

## 关键内容
- 售前阶段主要包括询价 `IN` 和报价 `QT`，可形成“询价 → 报价 → 标准订单 → 外向交货 → 开票”的文档链。
- 标准销售订单 `OR` 是最常用订单类型，通常后续交货类型为 `LF`，后续开票类型为 `F2`。
- 框架协议包括计划协议 `DS`、数量合同 `MK`、价值合同 `WK1/WK2`。
- 数量合同和价值合同通常通过释放订单参照执行，系统跟踪已释放数量、剩余数量或目标金额。
- 特殊订单类型包括现金销售 `CS`、急单 `SO`、贷项凭证请求、免费交货、后续免费交货、寄售补货和寄售发货。
- 销售订单处理需要关注文档流、交货冻结、开票冻结、拒绝原因、不完整性检查和销售订单履行监控。
- 配置层面涉及销售凭证类型、行项目类别、计划行类别、复制控制、定价过程、拒绝原因和贷项凭证请求工作流等对象。
