# 终端 Claude 知识库使用说明

这页用于让终端 Claude 更接近 Claudian / Obsidian 窗口的使用体验。使用者不需要知道知识库目录结构，也不需要自己指定文件路径，可以直接用自然语言提问。

## 使用者可以直接这样问

### FICO 操作类

- 如何创建财务客商？
- 如何创建 WBS？
- 供应商付款怎么做？
- 客户收款怎么处理？
- 采购发票怎么校验？
- 销售开票怎么冲销？
- 月结怎么做？
- 年结余额怎么结转？

### 事务码查询类

- `F-53` 是什么？
- `BP` 是干什么的？
- `CJ20N` 怎么用？
- `MIRO` 和 `MIR6` 有什么区别？
- `FAGL_FCV` 是做什么的？
- `ZFIR043` 是什么功能？

### 概念解释类

- 什么是客商主数据？
- 什么是 WBS 元素？
- GR/IR 重分类是什么意思？
- 什么是外币评估？
- 标准价和移动平均价有什么区别？
- MR21 和 MR22 有什么区别？

### 综合查询类

- 帮我查一下 FICO 月结流程。
- 帮我整理采购发票到付款清账流程。
- 帮我解释销售开票到客户收款的流程。
- 查一下知识库里有没有物料分类账相关内容。

## Claude 的检索策略

当用户提出 SAP / FICO / ABAP / MM / PP / QM 相关问题时，Claude 应按以下顺序检索知识库：

1. 先看 [[wiki/index]]，判断是否已有入口。
2. 判断所属模块，FICO 问题优先看 [[wiki/fico/overview]]。
3. 操作类问题优先查 `wiki/<module>/sources/`。
4. 事务码问题优先查 `wiki/<module>/entities/`。
5. 概念解释优先查 `wiki/<module>/concepts/`。
6. 跨主题问题再查 `wiki/<module>/synthesis/` 或多个来源页。
7. 如果没有找到对应页面，应明确说明“当前 wiki 尚未沉淀该主题”，再给出通用回答。

## 回答格式建议

### 操作类问题

建议结构：

1. 先给核心事务码。
2. 再说明业务目的。
3. 按步骤列出操作路径。
4. 单独列出注意点。
5. 最后列出参考页面。

示例结构：

```markdown
核心事务码：`BP`

用途：创建和维护财务客商主数据。

## 操作步骤
1. ...
2. ...

## 注意点
- ...

## 参考页面
- [[wiki/fico/sources/客商创建]]
```

### 事务码问题

建议结构：

1. 事务码是什么。
2. 用在什么业务场景。
3. 和哪些事务码/概念相关。
4. 参考页面。

### 概念问题

建议结构：

1. 一句话定义。
2. 为什么重要。
3. 简化流程或例子。
4. 常见注意点。
5. 参考页面。

## 面向普通用户的原则

- 用户不需要知道文件路径，Claude 应主动检索。
- 用户不需要知道 `sources/entities/concepts` 的区别，Claude 应自行判断。
- 回答要用中文，少用英文解释。
- 必要的事务码、表名、字段名保留英文或 SAP 原名。
- 如果知识库资料不足，不要编造；应说明当前知识库没有沉淀。
- 除非用户要求维护知识库，否则不要主动新增或修改页面。
