# 单篇来源页模板

这个模板用于把 `raw/` 里的单篇原始资料整理成 `wiki/<module>/sources/` 下的一页总结。

## 适用场景

- 一篇文章
- 一个 PDF
- 一个网页剪藏
- 一条整理价值较高的原始笔记

## 命名建议

建议文件名使用中文主题，尽量简短清楚，例如：
- `采购发票处理流程.md`
- `ABAP 内表循环优化.md`
- `应收账款清账规则.md`

## 模板内容

```markdown
---
module: fico
kind: source
updated: 2026-07-06
source_count: 1
tags: []
---

# 来源标题

## 来源信息
- 来源类型：article / pdf / web / note
- 原始位置：`raw/fico/article/202607/xxx.md`
- 语言：中文 / 英文 / 中英混合
- 适用模块：FICO / ABAP / MM / PP / QM

## 一句话结论
这篇资料最重要的结论是什么。

## 关键内容
- 要点 1
- 要点 2
- 要点 3

## 相关实体
- [[实体页 1]]
- [[实体页 2]]

## 相关概念
- [[概念页 1]]
- [[概念页 2]]

## 和现有知识的关系
- 新增了什么
- 修正了什么
- 和已有结论是否冲突

## 后续动作
- 是否需要补实体页
- 是否需要补概念页
- 是否需要补对比页
```

## 来源页命名规则

整理到 `wiki/<module>/sources/` 的页面，建议使用以下命名方式：

- **优先使用中文主题名**：文件名直接写出资料主题，方便检索和阅读。
- **尽量简短明确**：一眼能看出这页讲什么。
- **必要时加模块限定**：如果主题在多个模块中容易混淆，可在标题里补充模块名。
- **一篇来源对应一页**：一篇资料通常整理成一页来源页，不要把多篇资料硬塞进同一页。

### 示例
- `采购发票处理流程.md`
- `FICO 凭证号规则.md`
- `ABAP 内表循环优化.md`
- `应收账款清账规则.md`

## 使用原则

- `raw/` 不强制套这个模板。
- 只有当一份资料值得沉淀时，才整理成这类来源页。
- 这类来源页是 `wiki` 层内容，不是 `raw` 层内容。
